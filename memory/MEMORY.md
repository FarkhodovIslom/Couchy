# Архитектурные решения команды

## AuthService: почему JWT а не сессии
Решение принято 2026-03-10. Причина: микросервисная архитектура требует
stateless аутентификации. Сессии не работают при горизонтальном масштабировании.
Смотри ТЗ-047 раздел 3.2.

## UserService: зависимость от AuthService
UserService валидирует токены через AuthService напрямую.
Не ходить в БД за пользователем без валидации токена.
