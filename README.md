## 💡 О проекте

**Couchy** — это умный AI-агент, который знает всю кодовую базу, технические задания (ТЗ), бизнес-требования (БТ) и внутренние процессы команды в реальном времени. Он выступает в роли живой памяти команды с агентной логикой и обучает нового разработчика прямо во время работы, снимая нагрузку с Тимлида.

**Главная метрика проекта:** сокращение времени онбординга с 2 недель до 3 дней.

---

## 🛠️ Технологический стек

Проект организован в виде монорепозитория (**Turborepo**) с четким разделением слоев:

```
Couchy/ (Monorepo)
├── apps/
│   ├── web/          # Next.js (App Router) + Tailwind CSS (Фронтенд)
│   └── api/          # NestJS + Bun (Бэкенд)
├── packages/
│   └── shared/       # Общие TypeScript интерфейсы — единый источник правды
└── memory/           # Plain Markdown memory storage (OpenClaw-style)
```

| Слой | Технология |
|---|---|
| **LLM** | Gemini 3.5 Flash (`gemini-3.5-flash`) с динамическим уровнем `thinkingLevel` |
| **Long-Term Memory** | Plain Markdown + auto-flush (OpenClaw-style) |
| **Knowledge Graph** | SQLite (`better-sqlite3`) — хранение связей между сервисами, ТЗ и репортами |
| **Backend** | NestJS на быстром рантайме **Bun** |
| **Frontend** | Next.js 14 (App Router) с современной темной темой и Glassmorphism дизайном |

## ⚡ Быстрый запуск

### Требования
- Установленный **Node.js** (v18 или новее)
- Установленный **Bun** (v1.x) для запуска бэкенда

### 1. Установка зависимостей
Из корня проекта выполните команду:
```bash
npm install
```

### 2. Запуск проекта в режиме разработки
Запустите одновременно Next.js фронтенд и NestJS бэкенд одной командой:
```bash
npm run dev
```
Фронтенд запустится на: [http://localhost:3000](http://localhost:3000)  
Бэкенд запустится на: [http://localhost:3001](http://localhost:3001)

## 🎭 Сценарий для демо (4 минуты)

**Персонаж:** Алибек, junior backend dev, проводит первую неделю в проекте.

1. **Вход в онбординг**: Откройте интерфейс, введите *"Алибек"*, выберите роль *"Junior Backend"* и нажмите кнопку **Начать онбординг**.
2. **Интерактивный трек (Sidebar)**: В левой панели автоматически загрузится персональный трек обучения из 3 ключевых шагов:
   - *Изучить ТЗ-047*
   - *Разобрать JWT архитектуру*
   - *Зависимость UserService*
3. **Вопрос агенту**: Напишите в чат:
   > *"Почему в AuthService используется JWT а не сессии?"*
   - Агент мгновенно начнет стримить ответ (SSE).
   - Под ответом отобразятся связанные сущности из графа знаний: `[ТЗ-047, JWT_Decision]`.
4. **Обнаружение пробела (Gap Detection)**: Задайте этот же вопрос еще раз (или воспользуйтесь кнопкой сценария). На 3-й раз система автоматически задетектит пробел в онбординге, добавит в граф ноду `gap` и выдаст красное предупреждение в правой панели:
   > *⚠️ Выявлен пробел знаний (Gap Detection)*
5. **Проактивные оповещения (Proactive Alerts)**: Напишите в чат:
   > *"Я изменил UserService"*
   - Агент проанализирует граф связей и выдаст предупреждение в реальном времени:
   > *⚠️ Проверь зависимость с AuthService (ТЗ-047)*
